# Graph Flow Puzzle

Небольшая игра-головоломка о распределении дискретного ресурса по сети связанных узлов. Ваша цель — перемещать единицы ресурса вдоль рёбер графа, пока каждый узел не достигнет своего целевого значения.

## Как запустить

Установите HTTP-сервер и запустите проект:

npm install http-server -g
cd ./путь/к/файлам/
http-server -p 80


Затем откройте [http://localhost](http://localhost) в браузере.

## Связь с теорией графов

Эта игра построена на концепциях теории графов и сетевых потоков. Игровое поле представляет собой **неориентированный граф**: каждый узел — это вершина, а каждая линия между узлами — это ребро, позволяющее передавать ресурс.

### Свойства узлов

Каждый узел имеет:

- **Текущее значение** — количество содержащегося ресурса
- **Целевое значение** — требуемое количество ресурса
- **Набор соседей** — определяется рёбрами графа

### Задача балансировки

Основную задачу можно рассматривать как **задачу балансировки на графе**: начиная с начальной конфигурации, где весь ресурс сосредоточен в нескольких узлах, необходимо перераспределить его так, чтобы каждая вершина имела ровно своё целевое значение.

Это напоминает классические задачи:

- Достижение сбалансированного потока в сети с заданным спросом или предложением в каждом узле
- Удовлетворение ограничений на вершинах в потоковой сети
- Задачи о транспортировке ресурсов с ограничениями

### Игровая механика и графы

Ваши ходы соответствуют передаче единиц ресурса вдоль рёбер. Пространство состояний — это множество всех целочисленных распределений на вершинах, сохраняющих общее количество ресурса.

Различные уровни изменяют **топологию графа**, что влияет на:

- Множество достижимых конфигураций
- Сложность поиска последовательности ходов к сбалансированному состоянию
- Наличие узлов-бутылочных горлышек и мостов

### Образовательная ценность

Игра помогает развить интуицию относительно:

- Влияния структуры графа (степень узлов, циклы, узкие места) на возможные потоки
- Способов трансформации глобальной конфигурации через локальные ходы
- Моделирования распределения ресурсов и ограничений потока с помощью графов

## Уровни сложности

- **Уровень 1**: 8 узлов, простая топология
- **Уровень 2**: 10 узлов с узкими местами
- **Уровень 3**: 12 узлов, сложная структура связей

Попробуйте пройти все уровни с минимальным количеством ходов!
